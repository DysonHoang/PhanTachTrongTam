A

Lý Thuyết

1. Đồ thị dạng cây

Một đồ thị T = (V, E) vô hướng N đỉnh được gọi là cây khi và chỉ khi thỏa mãn một trong các điều kiện tương đương sau (Daisy Chain Theorem):

Định nghĩa tương đương
  • T không chứa chu trình đơn và có đúng N − 1 cạnh
  • T liên thông và mỗi cạnh của nó đều là cầu
  • Giữa hai đỉnh bất kỳ tồn tại đúng một đường đi đơn
  • T liên thông và có N − 1 cạnh

Các thuật toán cần biết trước: DFS / BFS, LCA (Lowest Common Ancestor), và các CTDL như Fenwick Tree, Segment Tree.

2

Centroid — Đỉnh Trọng Tâm

Định lý Jordan (1869)

Với mọi cây N đỉnh, luôn tồn tại ít nhất một đỉnh centroid: khi xóa đỉnh đó đi, mọi cây con còn lại đều có không quá ⌊N/2⌋ đỉnh.

Thuật toán tìm Centroid — O(N)

Bắt đầu từ bất kỳ đỉnh nào. Nếu có cây con với hơn N/2 đỉnh thì đi xuống rễ của cây con đó. Quá trình này luôn kết thúc và tìm được centroid:

C++void ReSubsize(int u, int p) {
    Tsize++; Subsize[u] = 1;
    for (int v : A[u])
        if (v != p && !Par[v]) {
            ReSubsize(v, u);
            Subsize[u] += Subsize[v];
        }
}

int GetCentroid(int u, int p) {
    for (int v : A[u])
        if (v != p && !Par[v] && Subsize[v] > Tsize/2)
            return GetCentroid(v, u);
    return u;
}
3

Cây Centroid

Từ cây gốc, ta xây dựng đệ quy một cây mới gọi là cây centroid:

  1. Tìm centroid C của cây hiện tại → C là gốc cây centroid.
  2. Xóa C, thu được nhiều cây con.
  3. Đệ quy tìm centroid của mỗi cây con → trở thành con của C.
  4. Lặp đến khi mọi đỉnh được xử lý.
4

Tính Chất Cây Centroid

01

Đầy đủ đỉnh

Cây centroid chứa tất cả N đỉnh của cây gốc — không mất thông tin nào.

02

Độ cao O(log N)

Cứ mỗi lần phân rã, kích thước giảm ít nhất một nửa → chiều cao tối đa log₂(N).

03

Tính chất đường đi

Đường đi A→B trên cây gốc luôn đi qua LCA(A,B) trên cây centroid.

⚠️ Lưu ý quan trọng

Cây centroid là cấu trúc "giả" — khoảng cách và đường đi trên cây centroid khác hoàn toàn với khoảng cách thực tế trên cây gốc. Khi tính dist(u, v) ta luôn tính trên cây gốc (thông qua LCA thực sự), không phải trên cây centroid.

Cài Đặt Cây Centroid

Biến cần thiết

// Tsize: số đỉnh của cây đang xét
// Subsize[u]: số đỉnh cây con gốc u
// Par[u]: cha của u trên cây centroid
int Tsize, Subsize[N], Par[N];

4 hàm cốt lõi

C++// Bước 1: Đếm lại kích thước cây con
void ReSubsize(int u, int p) {
    Tsize++; Subsize[u] = 1;
    for (int v : A[u])
        if (v != p && !Par[v]) {
            ReSubsize(v, u);
            Subsize[u] += Subsize[v];
        }
}

// Bước 2: Tìm centroid — đi xuống nhánh nào có > Tsize/2 đỉnh
int GetCentroid(int u, int p) {
    for (int v : A[u])
        if (v != p && !Par[v] && Subsize[v] > Tsize/2)
            return GetCentroid(v, u);
    return u;
}

// Bước 3: Phân rã đệ quy, xây dựng Par[]
void Decompose(int u, int p) {
    Tsize = 0;
    ReSubsize(u, 0);
    int C = GetCentroid(u, 0);
    Par[C] = (p == 0) ? C : p;           // gốc cây centroid: Par[C] = C
    for (int v : A[C])
        if (!Par[v]) Decompose(v, C);    // đệ quy các cây con
}

// Gọi trong main():
Decompose(1, 0);

Hàm tính khoảng cách (LCA)

Khoảng cách thực trên cây gốc giữa hai đỉnh u, v tính qua LCA:

// dist(u,v) = depth[u] + depth[v] - 2 * depth[LCA(u,v)]
int Dist(int u, int v) {
    return Depth[u] + Depth[v] - 2 * Depth[GetLCA(u, v)];
}

Áp Dụng — Bài Toán TREE1

TREE1 ★★★☆☆ N, M ≤ 2×10⁵

Cho cây N đỉnh. Đỉnh 1 tô màu đỏ, còn lại màu xanh. Có M truy vấn:

  • 1 V — Tô đỉnh V thành màu đỏ
  • 2 V — In khoảng cách từ V đến đỉnh đỏ gần nhất
INPUT
5 4 1 2 2 3 2 4 4 5 2 1 2 5 1 2 2 5
OUTPUT
0 3 2

Phân tích — Tại sao Centroid?

Phương pháp brute force O(N·M) quá chậm. Centroid cho phép xử lý mỗi truy vấn trong O(log N) nhờ tính chất: đường đi giữa bất kỳ hai đỉnh A, B đều qua một centroid chung trên cây centroid.

Ý tưởng

Lưu best[u] = khoảng cách ngắn nhất từ đỉnh đỏ gần nhất trong cây con của u ở cây centroid. Khi update hoặc query, ta đi lên các cha Par[] tối đa O(log N) bước.

Thao tác Update — O(log N)

C++void Update(int u) {
    best[u] = 0;          // u chính là đỉnh đỏ → khoảng cách tới bản thân = 0
    int cur = u;
    while (u != Par[u]) { // đi lên đến gốc cây centroid
        u = Par[u];
        best[u] = min(best[u], Dist(u, cur)); // dist tính trên cây GỐC
    }
}

Thao tác Query — O(log N)

C++int Getans(int u) {
    int cur = u;
    while (u != Par[u]) {
        u = Par[u];
        best[cur] = min(best[cur], Dist(cur, u) + best[u]);
    }
    return best[cur];
}
⏱ Độ phức tạp: O(N log N + M log N)

Bài Tập Ứng Dụng

Bài 1 — TREE2: Tô màu có thể đổi

TREE2 ★★★★☆

Giống TREE1 nhưng truy vấn 0 V đổi màu (đỏ→xanh hoặc xanh→đỏ). Nếu không có đỉnh đỏ nào in -1.

Vấn đề: Vì màu có thể đổi ngược lại, ta không thể dùng mảng best[u] lưu một giá trị nhất định mà phải lưu tập hợp các khoảng cách rồi lấy min.

Giải pháp: Dùng multiset<int> best[u] — khi thêm đỉnh đỏ insert giá trị, khi bỏ màu đỏ erase giá trị.

C++multiset<int> best[N];

void Update(int u) {
    Color[u] ^= 1;   // đổi màu
    if (Color[u]) best[u].insert(0);
    else          best[u].erase(best[u].find(0));
    int cur = u;
    while (u != Par[u]) {
        u = Par[u];
        int d = Dist(cur, u);
        if (Color[cur]) best[u].insert(d);
        else            best[u].erase(best[u].find(d));
    }
}

int Getans(int u) {
    int ret = INF, cur = u;
    if (best[u].size()) ret = *best[u].begin();
    while (u != Par[u]) {
        u = Par[u];
        if (best[u].size())
            ret = min(ret, *best[u].begin() + Dist(cur, u));
    }
    return (ret == INF) ? -1 : ret;
}
⏱ O(N log N + M log²N)

📎 Nộp bài: spoj.com/problems/QTREE5

Bài 2 — TREE3: Xếp hạng sĩ quan

TREE3 ★★★☆☆

Gán một trong 26 cấp bậc A–Z cho mỗi đỉnh sao cho: hai đỉnh cùng cấp bậc thì đường đi giữa chúng phải qua một đỉnh có cấp bậc cao hơn. In cách gán hoặc "Impossible!"

Nhận xét thiên tài: Tính chất này chính xác là tính chất của cây centroid! Chỉ cần gán cấp bậc của mỗi đỉnh = độ cao của nó trên cây centroid.

Vì độ cao cây centroid ≤ log₂(N) ≈ 17 với N = 10⁵, và 17 < 26 = số chữ cái → luôn có nghiệm!

C++int Getrank(int u) {
    if (u == Par[u]) return Ranked[u] = 0;    // gốc = cấp A
    if (Ranked[u] != -1) return Ranked[u];
    return Ranked[u] = Getrank(Par[u]) + 1;
}

// In kết quả:
for (int i = 1; i <= n; i++)
    cout << (char)(Getrank(i) + 'A') << " ";
⏱ O(N log N)

Bài 3 — Race: Đường đi độ dài K ít cạnh nhất

Race (IOI 2011) ★★★★★

Cây có trọng số. Tìm đường đi có đúng tổng trọng số K và đi qua ít cạnh nhất. In -1 nếu không tồn tại.

Ý tưởng chia để trị: Với mỗi centroid C, một đường đi có độ dài K qua C được chia thành hai phần: từ một đỉnh u → C và từ C → một đỉnh v, với tổng trọng số = K.

Ta dùng mảng D[i] = {gốc, số cạnh ít nhất} để lưu số cạnh min để đi từ một đỉnh đến C với trọng số đúng bằng i. Xử lý từng nhánh con tuần tự để tránh kết hợp hai đỉnh cùng nhánh.

⏱ O(N log N)

📎 Nộp bài: vnoi.info/LQDRACE

Bài 4 — TREE4: Đếm đường đi độ dài K

Codeforces 161D ★★★★☆

Đếm số cặp (u, v) có đường đi đúng bằng K cạnh (u < v).

Dùng D[i] = {gốc, số đỉnh}. Với mỗi centroid C, duyệt từng nhánh con: với khoảng cách w từ đỉnh trong nhánh đến C, ta cộng D[K-w].count (số đỉnh ở các nhánh trước đã đến C với khoảng cách K-w).

⏱ O(N log N)

📎 Nộp bài: codeforces.com/161/D

Bài 5 — TREE5: Đếm đỉnh trong khoảng ≤ L

TREE5 ★★★★★

Có M truy vấn (v, L): đếm số đỉnh u thỏa mãn dist(u, v) ≤ L.

Tiền xử lý hai vector cho mỗi đỉnh u:

  • ans[u] — các khoảng cách d(x, u) với x là con của u trong cây centroid (sắp xếp tăng dần)
  • cntPa[u] — các khoảng cách d(x, Par[u]) với x là con của u (để trừ trùng)

Query: đếm đỉnh trong ans[u] ≤ L, rồi với mỗi cha p: cộng số đỉnh trong ans[p] ≤ L - dist(u,p), trừ đi đỉnh trong cntPa[u] ≤ L - dist(u,p) để loại các đỉnh đã đếm.

int Getans(int u, long long L) {
    int ret = upper_bound(ans[u].begin(), ans[u].end(), L) - ans[u].begin();
    int cur = u;
    while (u != Par[u]) {
        u = Par[u];
        long long d = L - Dist(cur, Par[u]);
        int add = upper_bound(ans[u].begin(), ans[u].end(), d) - ans[u].begin();
        int sub = upper_bound(cntPa[cur].begin(), cntPa[cur].end(), d) - cntPa[cur].begin();
        ret += add - sub;
    }
    return ret;
}
⏱ O(N log N + M log²N)

Khi nào dùng Centroid Decomposition?
  • Bài toán cập nhật đỉnh và truy vấn khoảng cách giữa các cặp đỉnh
